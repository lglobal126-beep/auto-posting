## 프론트엔드 명세 (Next.js)

## 1. 개요

- **역할**: 사용자 로그인, 파일 업로드, 초안 미리보기/수정, 발행 트리거 등 UI 제공
- **프레임워크**: Next.js (React, TypeScript)
- **배포**: Vercel 무료 플랜
- **인증/데이터 연동**: Supabase JS SDK 사용

## 2. 페이지 구조

- `/` (홈/대시보드)
  - 최근 초안/발행된 포스트 목록 요약
  - “새 포스팅 만들기” 버튼

- `/login`
  - 이메일/비밀번호 로그인 폼
  - (선택) 회원가입 또는 관리자가 생성한 계정 안내

- `/settings/accounts`
  - 네이버 블로그 연동 상태 표시 및 연동/해제 버튼
  - 인스타그램 연동 상태 표시 및 연동/해제 버튼

- `/drafts/new`
  - 사진/영상/영수증 업로드 UI
  - 메모 입력 필드 (텍스트에어리어)
  - 운영시간/주차/편의시설 수동 입력 섹션
  - “AI로 초안 생성하기” 버튼

- `/drafts/[id]`
  - 생성된 초안 상세
  - 네이버 블로그 제목/본문 편집 영역
  - 인스타그램 캡션/해시태그 편집 영역
  - 발행 대상 선택(네이버/인스타 체크박스)
  - 즉시 발행 버튼, (추후) 예약 발행 옵션

- `/posts`
  - 발행된 포스트 목록
  - 각 항목에 네이버/인스타 링크 표시

## 3. 주요 UI 컴포넌트

- `Layout`
  - 상단 네비게이션 바 (로고, 메뉴, 사용자 메뉴)
  - 모바일 대응 햄버거 메뉴

- `FileUploader`
  - 이미지/영상/영수증 업로드 (Supabase Storage 직업로드를 위한 프리사인 URL 또는 SDK 사용)
  - 업로드 진행률 표시

- `DraftEditor`
  - 블로그 제목/본문 텍스트 에디터
  - 인스타 캡션/해시태그 입력 필드

- `AccountConnectCard`
  - 네이버/인스타 각각의 연결 상태, “연결하기/해제하기” 버튼

## 4. 상태 관리

- **전역 상태**
  - 로그인 유저 정보 (Supabase Auth 세션)
  - 연동 계정 상태 (간단한 fetch 후 캐싱)

- **페이지 단위 상태**
  - 초안 생성/수정 중인 데이터 (로컬 상태 + 저장 시 API 호출)
  - 업로드 진행 상태 (로딩 스피너, 에러 메시지 등)

## 5. API 연동 방식

- Supabase JS SDK로:
  - Auth (로그인/로그아웃/세션 관리)
  - Storage (파일 업로드)
  - 필요 시 단순 쿼리는 Supabase RPC 또는 Rest API 활용 가능

- 백엔드 FastAPI와는:
  - `/api/drafts`, `/api/publish` 등 HTTP 호출 (fetch/axios 등)
  - 인증 토큰(Supabase 세션의 access token)을 헤더에 포함

## 6. UX 고려사항

- 모바일 우선 레이아웃 (세로 스크롤 기준, 버튼 크기 여유 있게)
- 사진/영상 업로드 시 미리보기 썸네일 제공
- AI 초안 생성 중에는 명확한 로딩 인디케이터 표시
- 네이버/인스타 발행 성공 시 해당 링크로 바로 이동 가능한 버튼 제공
